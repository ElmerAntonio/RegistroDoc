PREGUNTA 2: ¿DÓNDE ESTÁN LAS CONTRASEÑAS?
===========================================

RESPUESTA: Están en el CÓDIGO VBA (en texto plano - PROBLEMA)

Ubicación de Contraseñas:
┌────────────────────────────┐
│ Módulo: RD_SEGURIDAD       │
│ Función: IniciarSesion()   │
│ Línea ~10-25               │
└────────────────────────────┘

Contraseñas Hardcodeadas (SIN ENCRIPTACIÓN):
─────────────────────────────────────────
admin      → admin123
docente    → docente123
director   → director123

⚠️ PROBLEMA DE SEGURIDAD:
  - Las contraseñas están en texto plano
  - Cualquiera que abra el código VBA las ve
  - NO es seguro para producción comercial

✗ ESTO NO CUMPLE ESTÁNDARES COMERCIALES
✗ NO PASA AUDITORÍA ISO


SOLUCIONES PARA SEGURIDAD REAL:
════════════════════════════════

OPCIÓN 1: Encriptación Local (Media Seguridad)
──────────────────────────────────────────────
Guardar contraseñas encriptadas en hoja oculta

Ventajas:
├─ Visible en VBA pero encriptadas
├─ Mejor que texto plano
└─ Compatible con Excel basado

Desventajas:
├─ Aún es vulnerable si alguien sabe desencriptar
└─ No es estándar comercial

Código necesario:
```vba
Function EncriptarPassword(pwd As String) As String
    ' Usar algoritmo SHA-256 o similar
    ' Guardar hash en celda oculta
End Function
```

OPCIÓN 2: Autenticación de Dominio (Mejor)
───────────────────────────────────────────
Usar credenciales de Windows (Active Directory)

Ventajas:
├─ Contraseñas administradas por IT de empresa
├─ No guardadas en Excel
├─ Cumple normas corporativas
└─ Auditoria automática por Windows

Código necesario:
```vba
Function ObtenerUsuarioWindows() As String
    ObtenerUsuarioWindows = Environ("USERNAME")
End Function
```

OPCIÓN 3: Base de Datos Externa (Profesional) ★ RECOMENDADO
───────────────────────────────────────────────────────────
Usar SQL Server o Access para autenticación

Ventajas:
├─ Contraseñas NUNCA en Excel
├─ Cumple ISO 27001 (Seguridad de Datos)
├─ Auditoría integrada
├─ Escalable a múltiples usuarios
├─ Encriptación de base de datos
└─ ESTÁNDAR COMERCIAL

Desventajas:
├─ Requiere SQL Server (costo)
├─ Más complejo de implementar
└─ Requiere IT para mantenimiento

OPCIÓN 4: OAuth / Single Sign-On (Enterprise)
──────────────────────────────────────────────
Integración con Microsoft Azure AD o Google Workspace

Ventajas:
├─ Estándar industrial
├─ No guardar contraseñas en sistema
├─ Auditoría de empresa
├─ Acceso desde cualquier dispositivo
└─ CUMPLE TODAS LAS NORMAS COMERCIALES

Desventajas:
├─ Requiere cloud
├─ Mayor complejidad
└─ Costo de infraestructura


PREGUNTA 3: ¿TIENE TODAS LAS ISO DE NIVEL INTERNACIONAL?
==========================================================

RESPUESTA: NO, SOLO TIENE ISO 9001 PARCIAL. FALTAN MUCHAS. ⚠️

ISO Implementadas Actualmente:
══════════════════════════════

✅ ISO 9001:2015 (PARCIAL)
   │
   ├─ Cláusula 8 (Operación): ✓ Procesos documentados
   │
   ├─ Cláusula 9 (Evaluación): △ Parcial
   │   └─ Falta: Mediciones de desempeño completas
   │
   └─ Cláusula 10 (Mejora): △ Parcial
       └─ Falta: Programa formal de mejora continua

⚠️ ESTADO: 60% implementado

ISO FALTANTES (CRÍTICAS PARA COMERCIO):
═════════════════════════════════════════

✗ ISO 27001:2022 (SEGURIDAD DE INFORMACIÓN - CRÍTICA)
  Nivel: ALTO
  Requerimiento: Contraseñas encriptadas, auditoría, backup
  Estado: NO EXISTE en sistema actual
  Impacto: SIN ESTO NO PUEDES VENDER A EMPRESAS

✗ ISO 27035 (GESTIÓN DE INCIDENTES DE SEGURIDAD)
  Nivel: ALTO
  Requerimiento: Plan de respuesta a breaches
  Estado: NO EXISTE
  Impacto: Necesario si quieres certificación

✗ ISO 22301 (CONTINUIDAD DEL NEGOCIO)
  Nivel: MEDIO
  Requerimiento: Plan de recuperación ante desastres
  Estado: NO EXISTE
  Impacto: Importante para clientes grandes

✗ ISO 19011 (AUDITORÍA)
  Nivel: MEDIO
  Requerimiento: Protocolo de auditoría interna
  Estado: PARCIAL (tienes auditoría de cambios)
  Impacto: Mejorable

✗ ISO 31000 (GESTIÓN DE RIESGOS)
  Nivel: MEDIO
  Requerimiento: Análisis de riesgos formal
  Estado: NO EXISTE
  Impacto: Importante para productos comerciales

✗ GDPR (Regulación Europea - Si vendes en EU)
  Nivel: CRÍTICO
  Requerimiento: Privacidad de datos, consentimiento
  Estado: NO IMPLEMENTADO
  Impacto: ILEGAL en Europa sin esto
