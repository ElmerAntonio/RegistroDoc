3️⃣  AUDITORÍA DE ACCESO (ISO 27001 A.12.4)
──────────────────────────────────────────

CADA INTENTO DE LOGIN REGISTRA:
  ├─ Usuario (quién)
  ├─ Acción (LOGIN_EXITOSO / LOGIN_FALLIDO)
  ├─ Detalles (motivo si falló)
  ├─ Fecha y Hora (timestamp exacto)
  ├─ Tabla afectada (SEGURIDAD)
  └─ Hoja: _Auditoria

EJEMPLO:
  admin    │ LOGIN_EXITOSO      │ ""                  │ 15/03/2026 09:30
  docente  │ LOGIN_FALLIDO      │ Contraseña incorrec │ 15/03/2026 09:31
  usuario  │ CAMBIO_CONTRASEÑA  │ "Contraseña actualiz│ 15/03/2026 09:32

✅ CUMPLE ISO 27001 A.12.4 - Logging


4️⃣  VALIDACIÓN DE COMPLEJIDAD (ISO 27001 A.10.2.3)
──────────────────────────────────────────────────

NUEVAS REGLAS DE CONTRASEÑA:

  ✓ Mínimo: 8 caracteres
  ✓ Debe incluir: Mayúscula + Minúscula + Número
  ✓ Recomendado: También símbolo (!@#$%^&*)
  
  Ejemplo VÁLIDO:
    Admin2026!Segura ✅
    Docente$Registro2026 ✅
    
  Ejemplo INVÁLIDO:
    123456 ❌ (solo números)
    abcdef ❌ (solo minúsculas)
    ABCDEF ❌ (solo mayúsculas)
    password ❌ (menor a 8 caracteres)

Función: ValidarComplejidadContraseña()

✅ CUMPLE ISO 27001 A.10.2.3

──────────────────────────────────────

Funciones nuevas:
  
  1. CrearRespaltoAutomatico()
     └─ Crea respaldo cada vez que se guarda
     └─ Ubicación: C:\Users\[Usuario]\Documents\RegistroDoc_Respaldos\
     └─ Nombre: RegistroDoc_YYYYMMDD_HHMMSS.xlsm
  
  2. ObtenerUltimoRespaldo()
     └─ Devuelve ruta del último respaldo
     └─ Permite recuperación rápida
  
  3. Política de retención:
     └─ Mantener últimos 30 respaldos
     └─ Eliminar respaldos mayores a 90 días
